Class

class is a blue print of an object . it defines the structure of object how the object is going to be look like. it is basically a collection of object or we can say that it is collection of methods and attributes.


Object

object is a real world entity. it is instance of class. memory is allocated to the attributes of class when its object is created.


Abstraction

It is a feature or property in which we shows only essential details or hide the background details or implementaion like if we create a stack we hide the implementation of it and we only show essential features like push, pull etc.


Data Hiding

Data hiding is common feature of object orientation in which we hide the data by providing access specifiers like public, private, protected, and default.


Encapsulation

Encapsulation is a process or way of binding object with it's attributes and behaviour. Like in real life we carry our cell phone with ourself it's attributes like camera , sensors , screen etc. are always with the phone and also it carries it behaviours of calling and other features along with it. So this method of wrapping of data in single unit or capsule is called as encapsulation.


Polymorphism

Basic definition of polymorphism is single name and multiple functionality. There is two types of polymorphism :-
  1. compile time
  2. run time

compile time polymorphism is function overloading and operator overloading while run time polymorphism is method overriding with virtual function.
